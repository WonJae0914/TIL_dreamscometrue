데이터 분석, 빅데이터

데이터 분석이란? 
데이터를 통해서 만들어 낼 수 있는 가치나 서비스를 제공하는 과정
데이터 분석 자체가 목적이 되는 경우는 거의 없다.

데이터는 비즈니스에서 윤활유 같은 역할을 한다. 

영역별 데이터가 제공할 수 있는 가치와 서비스 
 - 영업/CS/개발

1) 영업
신규 고객 차출, 기존 고객 유지 
데이터를 통해 신규 고객을 창출할 수 있는 기회와 영역을 탐색

2) CS
자동화 응대, 이슈 대응을 위한 데이터 활용

3) 개발
제품의 안정화의 버그 관리 

기획/UX
기획에서 ux개선뿐만 아니라 데이터에 기반해 새로운 서비스를 기획하는 것도 가능.
- 분류, 추천, 랭킹서비스등 데이터에 기반한 새로운 서비스 제공

마케팅 

분석 프로세스
문제정의 -> 데이터 수집 -> 데이터 처리 -> 분석 -> 리포팅/피드백

문제정의 
목적, 이유, 영향, 구체적설계, 지표, 일정, 예상, output
협의가 필요(데이터 분석가 <-> 데이터 엔지니어)
분석가는 요청자의 모호한 언어를 개발자들이 이해할 수 있는 언어로 바꿔줘야 한다.
요청자의 니즈를 정확히 파악하고 문제 정의 과정의 리딩이 필요하다.

데이터 수집
일반적으로 빅데이터 영역은 데이터 분석가, 데이터 엔지니어의 협업이다.
수집과 처리영역은 데이터 엔지니어의 역할이 큰 비중을 차지한다.

로그 데이터 수집 및 처리과정
 1) 로그 설계 
  - 로그의 항목과 포멧정의
  - 키, 밸류, 파라미터 정의
 2) 모듈화
  - 모듈화 개발
  - 모듈 적용 및 테스트
 3) 검증
  - 데이터의 퀼리티 검증과 관리
  - 수집 과정 모니터링

분석가는 설계와 검증을 담당하여 목적에 맞춰 무수한 로그들을 선별하고 항목을 정의한다.
데이터가 정의한대로 잘 쌓이고 있는가를 확인하고 수정

데이터 처리 

엔지니어vs분석가
 - 엔지니어 : 실시간, 시간대별 배치 작업을 통해 테이블을 업데이트 하거나 동기화하는 업무를 맡음
 - 분석가 : 데이터추출, 필터링, 그룹핑, 조인(SQL)
            DBMS, Hadoop으로 데이터 분석을 위한 기본적인 테이블을 만드는 단계
            테이블과 컬럼의 명칭, 처리/집계 기준, 조인시에 데이터 증식 방지가 필요
            엔지니어의 도움이 필요한 순간이 많다.
 - 이상치 제거, 분포변환, 표준화, 카테고리화, 차원축소
   -> 데이터 분석가가 주도적으로 진행하는 경우가 많다. 
   -> 재분석 진행시에는 이 과정으로 돌아와서 개선하는 경우가 많음

데이터 분석

 - 여러 상황에 따라 다양한 분석을 진행

 1. 지표 정의 및 트래킹
 - 비즈니스와 관련한 주요 지표를 개발/산출하고 대시보드 및 리포트를 통한 트래킹
 
 2. 탐색적 데이터 분석
 - 그룹별 평균, 합 등의 현황 확인
 - 분포 확인

 3. 통계 분석
 - 가설 검정, 모수 추정
 - 변수간 관계 파악과 변수간 영향력 파악
 - 통계 모형 구축
 - 요인분석, 군집분석

 4. 머신러닝
 - 분류 및 회귀문제 해결
 - 추천 및 이상치 탐지, 클러스터링 등(비지도 학습)

 리포팅 및 피드백

 분석결과를 설득력 있게 정리/전달
 1. 내용의 초점은 상대방
  - 상대가 이해할 수 있는 언어 사용
  - 상대의 입장과 니즈에 맞춰 생각하고 결과를 정리할 것
  - 목적을 수시로 상기하고 확인
  - 과정보다는 결과중심
 2. 간결하고 명확한 메세지
  - 짧은 문장(불필요한 수식어 X)
  - 핵심 내용의 전달이 우선 -> 근거
 3. 적잘한 시각화
  - 항목간 비교시에는 막대 그래프가 눈에 잘들어 오는 편
  - 변수간 관계는 산점도
  - 시계열은 라인으로 처리
  - 분포는 박스플롯
 4. 투명하고 공정한 피드백
  - 사실 기반의 공유/보고
  - 실패를 두려워하지 말 것
  - 빠르게 적극적인 대응필요 

Funnel 분석
 -> 유저들이 이탈하는 이유에 대해 조사

1. 유저들이 잘 사용하는 기능들
2. 개선점 확인
3. 결과는 언제까지?

새로운 시나리오대로 유저들이 잘쓰는가?
그렇다면 어떤 기능들을 잘쓰는가?
이탈이 많다면 어느 구간인가?
 -> 유저들이 남긴 데이터를 추적하여 그것을 기반으로 신규 시나리오 사용성을 파악
    (목적 : 유저 피드백을 통해서 러닝을 통해 전반적인 사용자 경험을 높여나간다)
 -> 행동 패턴의 파악이 필요
